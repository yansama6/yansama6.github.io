使用hexo+Github建博客

前言

我买的是腾讯云的服务器，已经跑起来一个多月了，才开始着手建立博客的，当时我是打算等这学期毕业再去弄这个博客，但是最近在跟着学校副院长成立了一个项目，已经开始架构前端了，目前前端已经到了收尾阶段，等前端都串连起来就把代码放上来，希望大家指出我们的问题！谢谢大家！我的邮箱

本教程只针对Linux用户

一、了解背景

既然要使用hexo和Github建博客首先要知道这两个东西是个什么玩意


hexo:

hexo是一个快速、简洁且高效的博客框架。Hexo使用Markdown解析文章，在几秒内，即可利用靓丽的主题生成静态网页。并且它是一款基于Node.js的静态博客框架，出名于依赖少、易于安装使用。

这里不再赘述Github，有兴趣的同学可以上这个网站，讲解非常清晰。Git廖雪峰教程据说Github是全球最大的同性交友网站
笑…

二、开始实战

打开终端首先建立一个文件夹，名字随意，推荐hexo为文件名字

mkdir hexo

建立好了之后进入此文件夹

cd hexo/

这里要安装一下

  • Node.js
  • Git
    这是搭建hexo的环境

    Git的安装教程在上面的廖雪峰的Git教程里面

sudo apt install npm

安装完成之后检测下环境是否安装完毕

如图可以看到版本号就表示安装成功

好了环境配好，只差软件

安装hexo

npm install -g hexo-cli

这个的安装时间有点长，而且它在查找下载的时候总给我一种不舒服的感觉。安装完成之后就是一些初始化问题

hexo init 项目名
cd 项目名

生成静态文件

hexo g

启动服务

hexo s

打开一个新的标签页，输入localhost:4000
接下来你会看到一个欢迎页面，顶端显示一个大大的Hexo

Hexo

我这里就不上图片了，如果看到了这个页面，恭喜你，这就表示项目创建成功并而成功跑起来了

接下来我们要配置Github与其链接，先在你的Github中建立一个名字为XXXX.github.io的库(XXXX为你的Github用户名，这是必须的)



建立了之后转到你的终端，进入你的项目之中，输入如下命令 视情况而定

ls

这其中有一个名字为 _config.yml 的文件，这是就是你的博客的配置文件。
用你的编辑器打开它！



向其中加入上图的代码保存！上图的yansama6是我的github账户名，这里替换成你的。
注意在冒号 “:” 之后要一个空格才可以生效
这个时候再执行以下命令

hexo clean
hexo g
hexo d

部署代码

npm install hexo-deployer-git --save
hexo d

输入你的Github账户名字和密码，这里我提一点，以后写了博客完成之后不要太兴奋急着提交，先启动服务，在本地看一下，是否符合心意，不要做无用功！

提交成功之后在url栏输入

https://XXXX.github.io

如果成功打开页面就是表示搭建博客成功，说通俗就是可以通过网络对其访问了，下面说说主题

三、主题配置

在你的项目目录下有一个theme的文件夹，在这里面放置你下载的主题，主题下载的话去google一个喜欢的就可以了，一般一个主题里面的内容如下图，当然也有可能有出入，都不影响


你会发现这里也有一个 _config.yml 文件，这个配置文件是你选择的主题的配置文件，这里面可以添加各种功能，或者删除功能，具体配置请参考主题制作者的github或者其博客，因为我个人喜欢简单的 那我在这里推荐两个简单的主题名字，如果喜欢比较骚的，就可以去google上搜索，有很多选择。

Next

这个是我用的网站主题，其中有3个样式，比较简单，我很喜欢。作者也有非常详细的配置教程

Maupassant

这个是另一个简洁的主题，也是一个不错的选择！

如果选择好了将其下载下来，把这个主题整个发到你的项目目录下的Theme目录中，然后返回上一层，打开 _config.yml 找到theme，它本身有一个默认的主题landscape，将这个landscape改为你的主题的文件的名字

保存退出，然后启动本地服务，你会看见主题的变化，如果出现乱码大部分原因是，主题文件缺失具体问题具体分析。

内容的修改在你的主题的 _config.yml 文件中，具体个性化配置去google就可以了，网上一大堆。

四、结语

好了，到这里基本配置就已经完毕了！如果对我的文章内容有什么疑问就发email给我！

<script type="text/javascript">
    $(document).ready(function () {
        var trigger = $('.hamburger'),
            overlay = $('.overlay'),
            isClosed = false;

        trigger.click(function () {
            hamburger_cross();
        });

        function hamburger_cross() {

            if (isClosed == true) {
                overlay.hide();
                trigger.removeClass('is-open');
                trigger.addClass('is-closed');
                isClosed = false;
            } else {
                overlay.show();
                trigger.removeClass('is-closed');
                trigger.addClass('is-open');
                isClosed = true;
            }
        }

        $('[data-toggle="offcanvas"]').click(function () {
            $('#wrapper').toggleClass('toggled');
        });
    });
</script>
